# 飞书开放平台后台填写清单

更新时间：2026.04.28

## 1. 先把概念说清楚

飞书开放平台负责的是“应用能力、机器人入口、菜单、权限、事件订阅、客户端内打开方式”。它不会自动托管我们仓库里的 Node.js / React / Agent 代码。

当前项目最合适的开发路线是：

```text
飞书群聊 / 单聊
  -> 机器人收到消息事件
  -> 长连接推给本地 lark-cli
  -> scripts/lark-event-bridge.mjs 转发到本地 API
  -> Agent 调豆包 2.0 Pro 规划和生成
  -> 飞书 Docs / Slides / 群消息交付
```

所以，当前阶段不需要公网 HTTPS。只要本地同时启动 API 和长连接桥接，就可以在飞书里真实触发 Agent。

后续如果要给评委远程体验、让手机离开本机网络也能稳定打开仪表盘，才需要把 Web 仪表盘或后端运行时部署到一个可访问地址。

## 2. 综合信息

截图位置：应用后台左侧第一个“综合信息”页面。

### 管理后台主页

当前建议：

```text
留空
```

原因：

- 这个字段是给“管理员从飞书后台进入你的管理页面”用的。
- 它不是机器人触发入口。
- 它也不是事件订阅地址。
- 当前项目还没有正式部署的管理后台 URL，填 `localhost` 对飞书移动端和其他用户没有意义。

什么时候填写：

- 后续把仪表盘部署成可访问页面后，再填部署后的 HTTPS 地址。

示例：

```text
https://your-domain.example.com/admin
```

## 3. 添加应用能力

截图位置：`添加应用能力`。

### 必须保留

| 能力 | 是否需要 | 用途 |
| --- | --- | --- |
| 机器人 | 必须 | 在群聊 / 单聊中接收消息、发送确认请求和交付摘要 |

### 当前推荐添加

| 能力 | 是否现在添加 | 用途 |
| --- | --- | --- |
| 网页应用 | 推荐后续添加 | 把当前 Web 仪表盘包装成飞书客户端内可打开页面 |
| 工作台应用 | 可选 | 在飞书工作台给用户一个进入仪表盘的应用入口 |

### 暂时不要加

| 能力 | 原因 |
| --- | --- |
| 工作台小组件 | 会增加前端适配成本，当前主链路不依赖 |
| 云文档小组件 | 适合嵌到云文档内扩展，当前主链路先不做 |
| 多维表格插件 | 与本赛题主链路关系不大 |
| 链接预览 | 不是 IM 到 Doc / PPT 的核心入口 |
| 消息快捷操作 | 后续可做“把这段消息整理成文档”，当前先不做 |
| 原生集成应用 | 当前项目不是原生桌面 / 移动 SDK 集成路线 |

## 4. 按场景添加

截图位置：`添加应用能力 -> 按场景添加`。

### 会话场景

当前只需要：

```text
发送和接收消息
```

这对应机器人能力，是 IM 入口的核心。

可选后续：

```text
聊天框“+”菜单
```

如果你希望用户在聊天输入框旁边点“+”，再打开一个页面或填充指令，可以后续加。当前通过 `@机器人`、`/agent` 和机器人自定义菜单已经够演示主链路。

### 工作台场景

当前可先不加。等仪表盘要做成飞书内可打开页面时，再加：

```text
工作台应用
```

## 5. 机器人配置

截图位置：左侧机器人图标。

### 如何开始使用

填写这句，控制在 64 字以内：

```text
在群聊中发送 /agent 或 @我，说出要整理的讨论，我会生成文档和PPT。
```

### 消息卡片回调请求方式

当前建议：

```text
暂不配置
```

原因：

- 当前项目主链路还没有用飞书交互式卡片按钮。
- 现在的确认、进度、取消都走普通文本消息。
- 等后续要做“卡片按钮确认 / 取消 / 重新生成”时，再去 `事件与回调 -> 回调配置` 配置。

### 机器人自定义菜单

建议开启：

```text
菜单状态：开启
展示形式：可切换菜单
默认展示：菜单
```

当前项目还没有接 `application.bot.menu_v6` 这类菜单事件，所以菜单的响应动作先不要选“推送事件”。建议用“发送文字消息”，让菜单复用已经跑通的文本触发逻辑。

推荐菜单：

| 菜单名称 | 响应动作 | 内容 |
| --- | --- | --- |
| 开始整理 | 发送文字消息 | `/agent 请整理当前群聊讨论，生成需求文档、5页汇报PPT和3分钟讲稿。` |
| 查看进度 | 发送文字消息 | `进度` |
| 取消任务 | 发送文字消息 | `取消` |

如果你想加“打开仪表盘”菜单：

| 菜单名称 | 响应动作 | 桌面端链接 | 移动端链接 |
| --- | --- | --- | --- |
| 打开仪表盘 | 跳转至指定链接 | 后续部署后的仪表盘 URL | 后续部署后的仪表盘 URL |

注意：`http://localhost:5173` 只对本机桌面调试有意义，飞书移动端和其他机器打不开。这个菜单最好等仪表盘有可访问地址后再填。

菜单配置后需要发布版本，通常几分钟后才会在飞书客户端生效。

## 6. 事件与回调

截图位置：左侧“事件与回调”图标。

### 事件配置

订阅方式选择：

```text
使用 长连接 接收事件
```

当前必须订阅的事件：

```text
接收消息 im.message.receive_v1
```

截图里已经有这些事件：

- 消息已读 `im.message.message_read_v1`
- 消息被 reaction `im.message.reaction.created_v1`
- 消息被取消 reaction `im.message.reaction.deleted_v1`
- 接收消息 `im.message.receive_v1`

当前项目只需要 `接收消息 im.message.receive_v1`。其他几个可以保留，也可以删掉以减少噪音；本地脚本默认只订阅 `im.message.receive_v1`，所以即使后台保留其他事件，也不会影响主流程。

点击“验证”前，先在本地运行：

```powershell
npm run dev:lark-events
```

如果没有启动长连接桥接，后台验证大概率会失败。

### 回调配置

当前建议：

```text
暂不配置
```

原因：

- 这里主要用于卡片按钮、交互回调等。
- 当前文本确认链路不需要它。

### 加密策略

当前长连接路线建议：

```text
Encrypt Key：留空
Verification Token：保留平台生成值，不要写入代码或文档
```

说明：

- 长连接模式由 `lark-cli` 和飞书官方 SDK 建立连接，本地不需要填写公网回调的加密配置。
- 如果后续切换到“将事件发送至开发者服务器”的 webhook 模式，再把 Verification Token / Encrypt Key 写入本地 `.env`，但不要提交 Git。

## 7. 安全设置

截图位置：左侧盾牌图标。

### 重定向 URL

当前建议：

```text
留空
```

原因：

- 这个字段用于 OAuth 免登授权码跳转。
- 当前机器人长连接接收消息不需要 OAuth 重定向。
- 后续做网页应用免登时再填。

后续网页应用免登示例：

```text
https://your-domain.example.com/auth/feishu/callback
```

### IP 白名单

当前建议：

```text
不要开启 / 留空
```

原因：

- 本地开发网络出口 IP 可能变化。
- 开启白名单后，`lark-cli` 或本地调试调用开放平台 API 可能被拦。

什么时候开启：

- 后续后端部署到固定服务器，并且确认出口 IP 固定后再开启。

## 8. 权限管理

当前至少确认这些权限已经开通并发布：

| 权限 | 用途 |
| --- | --- |
| 获取与发送单聊、群组消息 | 机器人收发 IM 消息 |
| 获取群组中用户 @ 机器人的消息 | 群聊 @ 机器人触发 |
| 获取群组中其他机器人和用户 @ 当前机器人的消息 | 群聊消息事件解析 |
| 读取用户发给机器人的单聊消息 | 单聊触发 |
| 云文档创建 / 编辑相关权限 | 创建和更新需求文档 |
| Slides 创建 / 读取 / 编辑相关权限 | 创建非空白演示稿 |

权限或事件修改后，一定要发布版本。只保存草稿不一定会在飞书客户端生效。

## 9. 在飞书里怎么触发

先本地启动两个终端：

```powershell
npm run dev
```

```powershell
npm run dev:lark-events
```

然后在飞书测试群里发送任意一种：

```text
/agent 请整理刚才关于校园活动报名系统的讨论，生成需求文档、5页汇报PPT和3分钟讲稿。
```

或：

```text
@机器人 请整理刚才讨论，生成需求文档、汇报PPT和3分钟讲稿。
```

后续控制：

```text
确认
进度
取消
```

如果配置了机器人菜单，也可以点击：

```text
开始整理
查看进度
取消任务
```

## 10. 当前最小可演示链路

```text
1. 飞书测试群发几条需求讨论。
2. 发送 /agent 指令或点击“开始整理”菜单。
3. 本地 dev:lark-events 收到真实飞书消息事件。
4. API 创建 Agent 任务。
5. Agent 请求确认。
6. 群里回复“确认”。
7. Agent 生成真实 Docs、Slides 和讲稿。
8. 群里收到交付链接。
9. 打开 Docs / Slides 验证内容非空。
10. Web 仪表盘作为辅助展示，不再当唯一最终产物。
```
