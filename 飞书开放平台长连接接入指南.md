# 飞书开放平台长连接接入指南

更新时间：2026.04.28

## 1. 结论

当前项目优先走飞书开放平台的长连接事件订阅，不需要公网 HTTPS。

两种事件接入方式的区别：

| 接入方式 | 是否需要公网 HTTPS | 适合场景 | 当前建议 |
| --- | --- | --- | --- |
| Webhook 回调 | 需要 | 线上部署、服务端固定域名 | 后续部署再做 |
| 长连接 / WebSocket | 不需要 | 本地开发、比赛演示、Agent 调试 | 当前优先 |

当前本地链路应改成：

```text
飞书测试群消息
  -> 飞书开放平台长连接事件
  -> lark-cli event +subscribe
  -> scripts/lark-event-bridge.mjs
  -> POST http://localhost:8787/api/lark/events
  -> Agent-Pilot 任务会话
  -> 飞书 Docs / Slides / 群回发
```

## 2. 飞书开放平台后台配置

按截图逐项填写时，优先参考：

```text
飞书开放平台后台填写清单.md
```

进入飞书开放平台：

```text
https://open.feishu.cn/app
```

打开已经创建的企业自建应用。

### 2.1 凭证与基础信息

确认：

- App ID 已存在。
- App Secret 已配置在本地 `lark-cli`，不要写进 Git。
- 应用处于可用状态。

本地检查：

```powershell
lark-cli doctor
lark-cli auth status
```

### 2.2 启用机器人能力

在应用后台进入：

```text
应用能力 -> 机器人
```

确认：

- 已启用机器人能力。
- 机器人名称和头像已配置。
- 机器人已被添加到测试群。

如果机器人没有进群，飞书不会把群消息推给这个应用。

机器人自定义菜单建议先使用“发送文字消息”，不要先接“推送事件”：

```text
一级菜单最多 3 个。推荐：
任务 -> 开始整理 / 查看进度 / 取消任务
仪表盘 -> 跳转 https://feishu-ai-im-collab-assistant.pages.dev
帮助 -> 发送使用说明
```

这样菜单可以复用当前已经实现的文本触发链路。

### 2.3 权限管理

至少需要确认以下能力：

| 能力 | 用途 |
| --- | --- |
| 接收消息事件 / `im.message.receive_v1` | 让应用收到群聊或单聊消息 |
| 发送消息 | Agent 回发确认请求和交付摘要 |
| Docs 创建 / 更新 | 生成和迭代需求文档 |
| Slides 创建 / 读取 | 生成演示稿并验证内容 |

如果 `lark-cli` 提示缺 scope，先用：

```powershell
lark-cli auth scopes
lark-cli auth login --recommend
```

必要时按提示重新授权。

### 2.4 事件订阅

进入：

```text
事件与回调 / 事件订阅
```

选择：

```text
使用长连接接收事件
```

添加事件：

```text
im.message.receive_v1
```

保存后，如果后台要求发布版本，则进入版本管理发布或提交审核。企业自建应用通常需要让新权限和事件订阅进入已发布版本，飞书客户端侧才会真正生效。

当前不需要配置：

- 管理后台主页。
- OAuth 重定向 URL。
- IP 白名单。
- Webhook 回调地址。
- 卡片回调地址。

这些入口只在后续做网页免登、远程部署或交互式卡片时再补。

## 3. 本地启动方式

开三个终端。

终端 1：启动 API + Web 仪表盘。

```powershell
npm run dev
```

终端 2：启动飞书事件长连接桥接。

```powershell
npm run dev:lark-events
```

该命令等价于：

```powershell
lark-cli event +subscribe --as bot --event-types im.message.receive_v1 --quiet
```

并把收到的事件转发到：

```text
http://localhost:8787/api/lark/events
```

终端 3：可选，用于检查 CLI。

```powershell
lark-cli event +subscribe --as bot --event-types im.message.receive_v1 --dry-run
```

## 4. 测试方式

在飞书测试群中发送：

```text
@Agent 请整理刚才关于校园活动报名系统的讨论，生成需求文档、5 页汇报 PPT 和 3 分钟讲稿。
```

预期结果：

1. `npm run dev:lark-events` 终端看到事件转发日志。
2. API 创建任务，状态进入 `planning`。
3. Agent 读取群上下文并生成计划。
4. 群里收到确认请求。
5. 回复：

```text
确认
```

6. Agent 继续生成 Docs、Slides、讲稿和交付摘要。
7. 群里收到交付链接。
8. Web 仪表盘同步显示任务步骤和产物链接。

可用的群内控制命令：

```text
确认
继续
进度
取消
```

## 5. 常见问题

### 5.1 `lark-cli event +subscribe` 要求 bot 身份

事件长连接只能用 bot 身份：

```powershell
lark-cli event +subscribe --as bot --event-types im.message.receive_v1
```

如果报 user identity 不支持，说明命令没有带 `--as bot`。

### 5.2 长连接启动了但收不到消息

逐项检查：

- 飞书后台是否选择“使用长连接接收事件”。
- 是否添加了 `im.message.receive_v1`。
- 应用版本是否已发布。
- 机器人是否已添加到测试群。
- 发送消息的群是否在 `LARK_ALLOWED_CHAT_IDS` 白名单中。
- 消息是否包含 `@Agent`、`/agent`、`请整理`、`生成需求文档` 等触发词。
- 是否被 `LARK_BOT_OPEN_ID` / `LARK_BOT_USER_ID` 识别为机器人自消息。

### 5.3 为什么还保留 HTTPS webhook 代码

长连接适合本地开发和演示；HTTPS webhook 适合后续部署到云端。当前代码已经支持 raw body 签名校验，后续如果切到 webhook，只需要配置公网 URL 和飞书后台回调即可。

### 5.4 现在还需要公网 HTTPS 吗

本地演示不需要。

比赛最终如果只展示本地运行 + 飞书长连接事件订阅，也可以讲清楚这是飞书开放平台官方支持的事件接收方式。为了让评委远程访问仪表盘，当前使用 Cloudflare Pages 固定入口；为了让 API 长期稳定，建议中期切换到 Named Tunnel + 自有域名或云服务器。

Named Tunnel 与长连接不冲突：飞书事件仍走长连接进入本地 `lark-cli`，仪表盘和 API 访问走稳定域名。只有后续切换到 webhook 时，才需要把事件接收方式改为 HTTPS 回调。

## 6. 当前推荐开发顺序

1. 先让 `npm run dev:lark-events` 收到真实群消息。
2. 确认消息能进入 `/api/lark/events` 并创建任务。
3. 确认群内“确认 / 进度 / 取消”可用。
4. 再优化 Web 仪表盘为飞书内嵌页面。
5. 最后做 Task / Event / Artifact 持久化恢复。
